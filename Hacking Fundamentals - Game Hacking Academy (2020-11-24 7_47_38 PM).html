Steps

All hacking consists modifying memory in a game. Writing any hack consists of four main steps:

  1. Identify what you want to change
  2. Understand what memory you need to locate
  3. Locate that memory in the game
  4. Change that memory

These steps apply for any hack, regardless of the complexity.

Identify

The first step for any hack is to identify what you want to do. This is self-explanatory, but it's important to have your end goal in mind so that you do not become overwhelmed. Different hacks will also require different approaches. For example, modifying a player's money will require a memory modification of a variable whereas seeing other players through walls will require a memory modification of the game's code.

Understand

To modify memory, we first need to locate it. Before we attempt to locate it, however, we need to understand what memory we need to locate. This is the hardest step and it requires you to understand how a particular hack actually works. In some cases, the memory you want to modify will be a variable. In other cases, you will want to modify large sections of code. There are three main types of modifications:

  • Variables, like modifying the value of a player's money
  • Code, like modifying how walls are drawn
  • Files, like modifying the saved items in your inventory

Locate

Once you know what you want to change and understand where to look for it, you can start looking for it. For some hacks, this may involve searching memory with a tool called a memory scanner. For others, it may involve looking through the game's code using a tool called a debugger. Depending on the game and approach, this can be a time-consuming process.

Change

When you have located the memory, the last step is to change it. Initially, this will mean using a memory scanner or debugger to manually modify the memory. Once you have verified this works, you can write a program to automatically change it for you.
